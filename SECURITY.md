# Security Policy

## Supported Versions

| Version | Supported |
|----------|-----------|
| Main Development Branch | Yes |
| Latest Release | Yes |
| Older Releases | Best Effort |

## Security Philosophy

TransportVault prioritizes:

- Confidentiality
- Integrity
- Availability
- Auditability
- Recoverability

## Reporting Vulnerabilities

Please report vulnerabilities privately through GitHub Security Advisories.

Do not publicly disclose vulnerabilities until coordinated remediation
has been completed.

## Security Scope

Security-sensitive areas include:

- Vault encryption
- Authentication
- Image verification
- Audit logging
- Installer execution
- Compliance reporting

## Compliance Statement

TransportVault may implement FIPS 140-oriented and NIST-aligned
architectures. Formal compliance or certification should not be assumed
unless explicitly documented.
